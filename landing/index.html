ทำไมต้อง SML Fleet?

เปรียบเทียบกับระบบ Fleet Management ทั่วไป — SML Fleet ให้ AI ทำงานแทนคน ลดต้นทุน เพิ่มประสิทธิภาพ

💰

ลดต้นทุนได้ทันที

  • ไม่ต้องจ้าง Dispatcher — AI fleet-ops จัดรถ จัดคนขับ มอบหมายงานให้อัตโนมัติ
  • ไม่ต้องจ้างคนตาม GPS — Movement Intelligence ตรวจจับรถ 24/7 ไม่ต้องนั่งเฝ้าจอ
  • ลดน้ำมัน — OSRM คำนวณเส้นทางจริงบนถนน หาเส้นทางสั้นที่สุด
  • ลดค่าซ่อม — AI แจ้งเตือนซ่อมตามรอบ ก่อนพังจริง (Preventive)
  • P&L ต่อคัน — รู้ทันทีว่ารถคันไหนกำไร คันไหนขาดทุน
⚡

เร็วกว่า — ทำงานทันที

  • พิมพ์แชท → AI ทำเลย — "มีรถ 6 ล้อว่างพรุ่งนี้ไหม" → ค้นหา + ตอบ + จอง ใน 5 วินาที
  • 1 นาที = ตรวจรถทั้งกอง — AI วิเคราะห์ GPS ทุกคันอัตโนมัติ ไม่ต้องเปิดแผนที่ดู
  • Alert ทันที — รถขับเร็วเกิน, ออกนอกเส้นทาง, วิ่งกลางคืน → แจ้ง Boss ทันที
  • ไม่มีเวลาทำการ — AI Workers ทำงาน 24/7 ไม่มีวันหยุด
  • สรุปรายวัน — CEO สรุป KPI + ปัญหา + คำแนะนำ ให้ทุกเช้า
🧠

ฉลาดกว่า — AI คิดเอง

  • ไม่ใช่แค่ rule-based — AI เข้าใจบริบท "90 กม./ชม. บนมอเตอร์เวย์ = ปกติ, ในเมือง = อันตราย"
  • monitoring_prompt — กำหนด AI rules ต่อรถแต่ละคัน "รถบรรทุกสารเคมี ห้ามเกิน 80"
  • Correlate ข้อมูล — "รถเพิ่งออกซ่อม + ขับผิดปกติ = น่าสงสัย ตรวจเบรค?"
  • สรุปภาษาไทย — AI เขียนรายงานภาษาคน ไม่ใช่แค่ตัวเลข
  • เรียนรู้จาก pattern — วิเคราะห์จากประวัติ ไม่ใช่แค่ข้อมูลปัจจุบัน

สถาปัตยกรรมระดับ Enterprise

Event-Driven + AI-First Architecture — ข้อมูลไหลตั้งแต่ Driver App จนถึง AI Analysis

Data Flow Architecture

📱 Driver App
GPS ทุก 30 วิ → ⚙️ Go API
Gin Framework → 🍃 MongoDB
Source of Truth → 📡 Kafka
Event Stream → 🐘 PostgreSQL
Query Layer → 🧠 HiClaw AI
วิเคราะห์ทุก 1 นาที

🍃 MongoDB → Kafka → PostgreSQL

Write Path: ทุก operation เขียน MongoDB ก่อน (Source of Truth) → Kafka produce event → Consumer upsert PostgreSQL
Read Path: ทุก query อ่านจาก PostgreSQL (เร็ว, JOIN ได้)
Rebuild: PostgreSQL สามารถ DROP แล้ว rebuild จาก MongoDB ได้ทุกเมื่อ — ข้อมูลไม่มีวันหาย

🧠 HiClaw AI Pipeline

Infinite Task: fleet-ops ทำงานทุก 1 นาที เรียก get_moving_vehicles ตรวจจับรถเปลี่ยนตำแหน่ง
AI Analysis: วิเคราะห์ 8 event types: speeding, geofence, night movement, erratic driving ฯลฯ
Auto Alert: severity warning/critical → Kafka → สร้าง alert ใน PostgreSQL อัตโนมัติ

🔒 Multi-tenant + Security

shop_id: ทุก document, ทุก table, ทุก query มี shop_id — แยกข้อมูลลูกค้าสมบูรณ์
JWT Auth: Token-based authentication + role-based access (admin, driver, manager)
Higress Gateway: AI requests ถูกกรองผ่าน Gateway — consumer auth, rate limit, token control

🛣️ OSRM Routing Engine

Road-following polyline: เส้นทางตามถนนจริง ไม่ใช่เส้นตรง — วาดบน OpenStreetMap
Turn-by-turn: คำแนะนำเลี้ยวทุกจุด ภาษาไทย สำหรับ Driver App
Upgrade path: พร้อมอัปเกรดเป็น Valhalla self-hosted (map matching, isochrone, truck profile)

📡 11 Kafka Topics + Real-time
